# YouTube Auto-Clip Bot - Complete Termux Guide\n\nFull step-by-step guide to run the bot on Android using Termux.\n\n## 📱 What is Termux?\n\nTermux is a terminal emulator for Android that provides a Linux environment. You can run the YouTube Auto-Clip Bot directly on your phone!\n\n## 🚀 Installation Guide\n\n### Step 1: Install Termux\n\n1. Download **Termux** from:\n   - [Google Play Store](https://play.google.com/store/apps/details?id=com.termux)\n   - Or [F-Droid](https://f-droid.org/packages/com.termux/)\n\n2. Open Termux after installation\n\n3. Grant storage permissions:\n   ```bash\n   termux-setup-storage\n   ```\n   - This will ask permission to access your phone's storage\n   - Accept the permission\n\n### Step 2: Update Termux Packages\n\n```bash\n# Update package manager\npkg update\n\n# Upgrade all packages\npkg upgrade\n```\n\n### Step 3: Install Required System Packages\n\n```bash\n# Essential build tools\npkg install build-essential\n\n# Python 3.11\npkg install python python-pip\n\n# Video processing\npkg install ffmpeg\npkg install opencv\n\n# Audio processing\npkg install libffi libcrypt\n\n# Git (to clone the repo)\npkg install git\n\n# Text editor (optional but useful)\npkg install nano\n```\n\n**This may take 10-20 minutes. Let it complete!**\n\n### Step 4: Clone the Repository\n\n```bash\n# Navigate to storage\ncd ~/storage/downloads\n\n# Clone the bot\ngit clone https://github.com/sanjay9867/youtube-auto-clip-bot.git\n\n# Enter the directory\ncd youtube-auto-clip-bot\n```\n\n### Step 5: Create Python Virtual Environment\n\n```bash\n# Create virtual environment\npython -m venv venv\n\n# Activate it\nsource venv/bin/activate\n```\n\n**You should see `(venv)` at the start of your terminal line**\n\n### Step 6: Install Python Dependencies\n\n```bash\n# Upgrade pip\npip install --upgrade pip\n\n# Install requirements\npip install -r requirements.txt\n```\n\n**⚠️ This will take 30-60 minutes. Be patient!**\n\nIf you encounter errors with specific packages:\n\n```bash\n# Try installing one by one\npip install numpy==1.24.3 -v\npip install scipy==1.11.0 -v\npip install opencv-python==4.8.0 -v\npip install librosa==0.10.0 -v\npip install zhipu-ai==2.1.5.20240606 -v\npip install google-api-python-client==1.10.0 -v\n```\n\n### Step 7: Run Setup Script\n\n```bash\n# Run setup\npython setup.py\n```\n\nThis will create directories and .env file.\n\n### Step 8: Configure API Keys\n\n```bash\n# Open .env file\nnano .env\n```\n\nAdd your API keys:\n\n```env\n# GLM 4 API Key (Required)\nGLM_API_KEY=your_glm_api_key_here\nGLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4\n\n# YouTube API Key (Optional, for upload)\nYOUTUBE_API_KEY=your_youtube_api_key_here\n\n# Settings\nMAX_WORKERS=2\nUSE_GPU=false\n```\n\n**To save and exit nano:**\n- Press `Ctrl + X`\n- Press `Y`\n- Press `Enter`\n\n### Step 9: Test Installation\n\n```bash\n# Activate virtual environment\nsource venv/bin/activate\n\n# Test GLM 4 connection\npython main.py --test-glm\n```\n\nYou should see: ✅ **\"GLM 4 connection successful!\"**\n\n---\n\n## 🎬 How to Use\n\n### Basic Usage\n\n```bash\n# Activate virtual environment first\nsource venv/bin/activate\n\n# Process a local video\npython main.py ~/storage/downloads/my_video.mp4 --auto-clip --auto-title\n```\n\n### Download from YouTube and Process\n\n```bash\npython main.py \"https://youtube.com/watch?v=dQw4w9WgXcQ\" --auto-clip --auto-title\n```\n\n### Without Clip Detection (Full Video)\n\n```bash\npython main.py video.mp4 --no-auto-clip --auto-title\n```\n\n### Generate Only Titles (No Clips)\n\n```bash\npython main.py video.mp4 --auto-title --no-auto-clip\n```\n\n### Debug Mode (Detailed Output)\n\n```bash\npython main.py video.mp4 --debug --auto-clip --auto-title\n```\n\n---\n\n## 🗂️ File Organization\n\n```\nTermux Storage:\n~/storage/\n├── downloads/\n│   └── youtube-auto-clip-bot/     ← Your project\n├── movies/\n│   └── (Input videos here)\n└── documents/\n    └── (Output clips here)\n```\n\n### Process Videos from Downloads\n\n```bash\ncd ~/storage/downloads/youtube-auto-clip-bot\n\n# Process video from ~/storage/movies/\npython main.py ~/storage/movies/myvideo.mp4 --auto-clip --auto-title\n```\n\n---\n\n## 🔧 Troubleshooting\n\n### Problem: \"command not found: python\"\n\n**Solution:**\n```bash\npkg install python\npython --version\n```\n\n### Problem: \"pip: command not found\"\n\n**Solution:**\n```bash\npkg install python-pip\npip --version\n```\n\n### Problem: \"Permission denied\" on storage\n\n**Solution:**\n```bash\ntermux-setup-storage\n# Grant permission when prompted\n```\n\n### Problem: \"ModuleNotFoundError: No module named 'cv2'\"\n\n**Solution:**\n```bash\nsource venv/bin/activate\npip install --upgrade opencv-python\n```\n\n### Problem: \"GLM API key error\"\n\n**Solution:**\n1. Check `.env` file: `cat .env`\n2. Verify your GLM API key is correct\n3. Get API key from: https://open.bigmodel.cn/\n\n### Problem: \"Out of memory\" error\n\n**Solution:**\n```bash\n# Edit config.yaml\nnano config.yaml\n\n# Change these settings:\nmax_workers: 1          # Reduce workers\nquality: \"720p\"        # Lower video quality\nbatch_size: 4          # Reduce batch size\n```\n\n### Problem: Installation takes too long\n\n**Solution:**\n- Termux runs on mobile hardware - expect 1-2 hours for full setup\n- Use WiFi, not mobile data\n- Keep phone plugged in\n- Don't close Termux during installation\n\n---\n\n## 📊 API Keys Setup\n\n### Get GLM 4 API Key (Required)\n\n1. Visit: https://open.bigmodel.cn/\n2. Sign up / Login\n3. Go to \"API Keys\"\n4. Create new API key\n5. Copy and paste to `.env`:\n   ```\n   GLM_API_KEY=your_key_here\n   ```\n\n### Get YouTube API Key (Optional)\n\n1. Visit: https://console.cloud.google.com/\n2. Create new project\n3. Enable YouTube Data API v3\n4. Create OAuth 2.0 credentials\n5. Download credentials.json\n6. Add key to `.env`:\n   ```\n   YOUTUBE_API_KEY=your_key_here\n   ```\n\n---\n\n## 💡 Performance Tips\n\n### For Termux (Mobile Device):\n\n```yaml\n# config.yaml optimizations\nvideo_processing:\n  quality: \"720p\"              # Not 1080p\n  fps: 24                       # Lower than 30\n  bitrate: \"2M\"                # Lower bitrate\n\nclip_detection:\n  scene_threshold: 0.4          # Less sensitive\n  min_clip_duration: 10         # Longer clips\n  max_clip_duration: 60         # Shorter max\n  \nperformance:\n  max_workers: 1                # Single worker\n  gpu_enabled: false            # No GPU on phone\n  batch_size: 4                 # Small batch\n```\n\n### Processing Speed:\n- 1 minute video: 5-10 minutes processing\n- 10 minute video: 30-60 minutes processing\n- Video quality & phone specs affect speed\n\n---\n\n## 📝 Example Workflows\n\n### Workflow 1: Process Local Video\n\n```bash\n# 1.
